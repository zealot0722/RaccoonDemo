# Fuzzy Semantic Test Rounds

## 目的

針對目前 demo 已支援的客服語意項目，設計五輪不重複模糊語意題目，驗證分類、上下文延續、追問、轉人工與結束流程是否穩定。

## 涵蓋項目

- FAQ
- 有效商品推薦
- 商品條件不足追問
- 商品上下文追問
- 查貨態命中
- 查貨態缺少編號
- 退貨申請缺少資料
- 退貨申請資料齊全
- 轉真人客服
- 客訴
- 閒聊
- 亂碼或低價值輸入
- 對話結束
- 複合意圖

## 通過標準

- 五輪共 75 題，使用者輸入不可重複。
- 每輪都要涵蓋全部 14 個項目。
- 不得把查貨態誤判為商品推薦。
- 不得把退貨售後語句誤判為查貨態。
- 不得把已知招呼語誤判為亂碼。
- 複合意圖應拆解可處理部分逐步回覆；只有包含明確真人客服、客訴或必要人工判斷時才進入 `needs_review`。

## 本輪發現並修正的盲點

- `包裹少件要處理` 原先會因為 `包裹` 被視為貨態訊號，與退貨形成複合意圖後轉人工；已改為只有包裹搭配 `到哪`、`在哪`、`位置`、`進度`、`還沒到` 等物流語意時才算查貨態。
- `hello` 原先會先命中英文亂碼規則而成為 `unclear`；已改為先判斷已知招呼語，再判斷亂碼。
